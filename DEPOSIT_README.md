# Chromophore rotational-space scan of the fluorescent-protein family

Code and data accompanying:

> Zimmer, M. *Ground-state chromophore geometry tracks quantum yield across
> fluorescent proteins, while cage size reports chemistry.* (submitted to
> *Biophysical Journal*).

A purely structural, reproducible analysis of how much rotational space the
chromophore has inside the beta-barrel of every GFP-like fluorescent protein in
the PDB (838 crystal structures with a rotatable methine bridge, derived from the
908-FP survey of Begg et al.).

## What is in this deposit

```
scripts/        all analysis code (Python 3.10; numpy, scipy, statsmodels, gemmi)
data/           curated and derived tables (the reproducibility core; see MANIFEST_deposit.txt)
DEPOSIT_README.md, MANIFEST_deposit.txt
```

The two primary machine-readable result tables are:

- `data/supplementary_SD1_per_structure.csv` -- one row per crystal structure (838),
  with every scan metric, geometry, B-factor, contact, barrel-shape, spectral, and
  curated-QY column.
- `data/supplementary_SD2_per_unique_fp.csv` -- one row per unique fluorescent
  protein (164; replicate crystals collapsed by median).

## NOT included here (large or externally available)

- Raw crystal structures (`data/cif/`, ~620 MB): downloaded on demand from RCSB
  by `scripts/scan_all.py`; not redistributed.
- Per-structure scan grids (`data/scans/`, `data/scans_25deg/`): regenerated by
  `scripts/scan_all.py` / `scan_all_25deg.py`. Available as a separate large
  archive on request.
- ESP intermediates (`data/pqr/`, ~400 MB): regenerated by the pdb2pqr pipeline
  (`scripts/validate_esp_apbs.py`); not central to the headline results.

## Reproducing the headline numbers

The fastest validation is the independent verifier, which recomputes every
headline statistic from the rawest tables and prints disk-vs-manuscript:

```
python scripts/verify_manuscript_numbers.py
```

The confirmatory robustness analyses (permutation p, Theil-Sen, jackknife,
outlier grid, phi/tau decomposition) and the heteroatom-exclusion control:

```
python scripts/eval_confirmatory.py          # -> data/eval_confirmatory.csv
python scripts/heteroatom_sensitivity.py      # -> data/heteroatom_sensitivity.csv
```

Full pipeline (from CIFs to tables), in order:

1. `scan_all.py`            full 2-D (tau, phi) scan -> `scan_all_summary.csv`, `data/scans/`
2. `scan_1d.py`            1-D I-bond / P-bond scans -> `scan_1d_summary.csv`
3. `curate_qy_from_fpbase.py`, `recover_qy_by_sequence.py` -> `lit_qy_curated.csv`
4. `canonical_cd_twist.py` -> `d_exp_canonical.csv` (headline resting twist)
5. `aggregate.py`          merge + per-class stats + figures -> `merged_for_aggregate.csv`
6. `gatekeeper_analysis.py` -> `gatekeeper_*.csv`
7. `count_hb_contacts.py`, `matched_pairs.py`, `loocv_validation.py`,
   `bootstrap_cis.py`, `twist_decomposition.py`, `eval_confirmatory.py`,
   `heteroatom_sensitivity.py`
8. `build_supplementary_csv.py` -> `supplementary_SD1/SD2`
9. `verify_manuscript_numbers.py` (validation)
